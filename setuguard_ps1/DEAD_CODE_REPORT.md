## Finding 1 — unused local variable `d` (DEX list) in `static_analysis.py:203`

```python
def analyze_apk(apk_path: str) -> dict:
    a, d, dx = AnalyzeAPK(apk_path)
```

`AnalyzeAPK()` returns a 3-tuple `(a, d, dx)` — the APK object, the DEX list, and the
Analysis object. `a` and `dx` are used throughout the rest of the function; `d` is bound
and never referenced again anywhere in `analyze_apk` or the rest of the file (confirmed:
`grep -n '\bd\b' static_analysis.py` matches only the assignment line itself).

**Proposed fix (for sign-off, not applied):** rename the binding to `_` —
`a, _, dx = AnalyzeAPK(apk_path)` — to make the non-use explicit. This is a one-line,
behavior-preserving rename, not a logic change.

**Why this isn't a bigger deal:** it costs nothing at runtime (holding a reference to the
DEX list briefly), so it's cosmetic/lint-level, not a correctness issue. Flagging per the
frozen-file sign-off rule rather than judging it in scope for a "measurement only" pass.
